# Beta — Итерации

> **Цель фазы**: полная игра с мета-прогрессией, обучением,
> juice-эффектами, балансом. Готовность к внешним плейтестам.
>
> **Предусловие**: Alpha завершена — полный контент, стабильный цикл.
>
> **Длительность**: 4–5 недель (7 итераций × 3–4 дня)

---

## B1 — Полные combo и модификаторы

**Цель**: все 7 combo и все 6 модификаторов комнат работают.

### Фичи

1. **Оставшиеся 4 combo**
   - Огонь + Лёд → Пар (ослепление всех в зоне)
   - Молния + Огонь → Огненная буря (усиленный AoE)
   - Кислота + Вода → Кислотный потоп (все клетки воды наносят урон)
   - Яд + Огонь → Токсичный дым (3×3 облако DoT + ослепление)
   - Обновить `DamageService.CheckCombo` для всех пар

2. **Оставшиеся 3 модификатора**
   - ⏳ Таймер: через 6 ходов — 99 урона всем.
     Счётчик на экране (большие цифры), пульсация при ≤2
   - 🌑 Тёмная: видимость 2 клетки от игрока, остальное —
     чёрный fog of war (шейдерная маска)
   - 🔄 Конвейер: все сдвигаются на 1 клетку в указанном
     направлении. Стрелки на полу

3. **Индикация модификаторов**
   - Иконка на узле карты (видна до входа)
   - Текст-подсказка при входе в комнату
     («Эта комната затоплена!» — 1.5 сек, fade out)

### Приёмочный тест (Play Mode)

```
1. Бросить Пламя в зону льда → «COMBO! Пар» →
   ослепление всех в зоне
2. Бросить Молнию в зону огня → «COMBO! Огненная буря» →
   увеличенный AoE + урон
3. Комната ⏳: счётчик «6» виден, каждый ход −1,
   при 0 → все получают 99 урона
4. Комната 🌑: видны только 2 клетки вокруг игрока,
   остальное чёрное (но враги по краю видимости — видны)
5. Комната 🔄: после каждого хода все сдвигаются вправо (стрелки),
   враг у стены — не двигается
6. На карте: модификатор виден как иконка рядом с узлом ⚔️
```

## B2 — Мета-прогрессия: рецепты и реликвии

**Цель**: рецепты и реликвии разблокируются через ачивки.
Начальный пул ограничен.

### Фичи

1. **AchievementService и AchievementTracker**
   - `AchievementTracker`: подписка на игровые события,
     обновление прогресса в `MetaProgressionModel`
   - Условия: `AccumulateStat`, `SingleRunCondition`, `CombatCondition`
   - При выполнении → разблокировка + нотификация + сохранение

2. **Разблокировка рецептов (7 ачивок)**
   - Стартовые: Пламя, Поток, Отрава, Пар, Напалм, Кислота
   - Молния: «Убей 3 врагов зельем воды за 1 ран»
   - Вспышка: «Дойди до Этажа 2»
   - Проклятие: «Убей элитного без урона»
   - Бездна: «Толкни 5 врагов»
   - Спора: «Используй фонтан 3 раза»
   - Миазма: «Нанеси ядом 30 урона»
   - Хаос: «Открой все рецепты»
   - Неразблокированные рецепты: merge даёт «неизвестное зелье»
     (визуально — серая колба с «?»), не попадает в инвентарь

3. **Разблокировка реликвий (5 оставшихся)**
   - Мутаген, Декантер, Шип, Вихрь, Монокль
   - Каждая привязана к ачивке (по GDD)
   - Неразблокированные не появляются в пуле выбора после элиты

### Приёмочный тест (Play Mode)

```
1. Новый профиль (сброс прогресса)
2. Доступно: Пламя, Поток, Отрава, Пар, Напалм, Кислота
3. Merge 🔵+🟡 → серая колба «?» (Молния не разблокирована)
4. Убить 3 врагов Потоком за ран → нотификация
   «Рецепт разблокирован: Молния!»
5. В следующем ране 🔵+🟡 → Молния lv1 (работает)
6. Книга рецептов: разблокированные — с описанием,
   остальные — «?» с условием разблокировки
7. Победить элиту → выбор реликвий только из разблокированных
8. Разблокировать Декантер → зоны длятся +1 ход (проверить)
```

## B3 — Мета-прогрессия: улучшения

**Цель**: постоянные улучшения разблокируются и применяются.
Экран ачивок функционален.

### Фичи

1. **Постоянные улучшения (5 штук)**
   - 5-й слот зелий: «Победи финального босса»
     → InventoryModel.maxSlots = 5
   - Расширенная доска 5×4: «Используй все 5 элементов за ран»
     → BoardModel.width = 5
   - 4-е действие: «Создай по 1 зелью каждого рецепта»
     → maxActions = 4
   - Стартовое зелье: «Дойди до этажа 3 три раза»
     → при старте рана — случайное зелье lv1 в инвентаре
   - Дальнозоркость: «Пройди 10 событий»
     → на карте видно на 2 ряда вперёд

2. **Экран ачивок**
   - `AchievementScreen`: список ачивок по категориям
     (Рецепты, Реликвии, Улучшения)
   - Каждая ачивка: название, описание, прогресс-бар,
     награда, статус (✓ / в процессе)
   - Доступ из меню (кнопка 🏆)

3. **Применение улучшений при старте рана**
   - `RunFactory.CreateRun(MetaProgressionModel)`:
     проверить unlocked upgrades, применить
   - 5-й слот: UI адаптируется (5 слотов вместо 4)
   - 5×4 доска: BoardView адаптируется, генерация с width=5
   - 4 действия: 4 точки вместо 3

### Приёмочный тест (Play Mode)

```
1. Открыть экран ачивок → 3 категории,
   прогресс отображается
2. Победить финального босса →
   «Улучшение: 5-й слот для зелий!»
3. Следующий ран → 5 слотов внизу экрана
4. Разблокировать расширенную доску → перегонка 5×4 (20 ячеек)
5. Разблокировать 4-е действие → 4 точки действий
6. Разблокировать стартовое зелье →
   ран начинается с 1 зельем в инвентаре
7. Разблокировать дальнозоркость →
   на карте видны узлы на 2 ряда вперёд
8. Все улучшения сохраняются между сессиями
```

## B4 — Обучение

**Цель**: первый ран — обучающий. 5 шагов без текстовых стен.

### Фичи

1. **TutorialController и TutorialOverlay**
   - `TutorialController`: FSM обучения, 5 шагов
   - `TutorialOverlay`: затемнение экрана кроме целевой области,
     анимированная стрелка, пульсирующая подсветка
   - `InputFilter`: блокировка ввода вне целевой области

2. **5 шагов обучения**
   - Шаг 1 (Merge): доска только 🔴 и 🔵, стрелка 🔴→🔴.
     Игрок мержит. Затем 🔵→🔵. Зелья в слотах
   - Шаг 2 (Первый бой): 4×4 сетка, 1 скелет.
     Подсветка: «тапни зелье → тапни клетку». Убить
   - Шаг 3 (Кулдаун и движение): 2 скелета. Бросок → кулдаун,
     подсказка второго зелья. Свайп для движения
   - Шаг 4 (Рецепт и подпитка): 🔴, 🔵, 🟢 на доске.
     Стрелка 🔴→🟢 (рецепт Напалм). Стрелка 🔴→варево (подпитка).
     Книга рецептов мигает
   - Шаг 5: свободная игра, карта этажа без подсказок

3. **Условия пропуска**
   - Если `MetaProgressionModel.tutorialCompleted == true` →
     обычный ран
   - Кнопка «Пропустить обучение» (мелкий текст в углу)
   - По завершению: tutorialCompleted = true, сохранить

### Приёмочный тест (Play Mode)

```
1. Сброс прогресса → новый профиль
2. «Старт» → обучающая доска (только 🔴 и 🔵)
3. Стрелка от 🔴 к 🔴 → перетащить → merge →
   «Пламя lv1» в слоте (подсветка)
4. Стрелка от 🔵 к 🔵 → merge → «Поток lv1»
5. Автопереход в бой → 1 скелет, 4×4 сетка
6. Подсветка: «тапни зелье» → тап → «тапни клетку» →
   тап → скелет умирает
7. Перегонка → бой 2: 2 скелета, кулдаун после первого броска,
   подсказка движения (свайп)
8. Перегонка → 🔴, 🔵, 🟢: стрелка 🔴→🟢 → Напалм,
   стрелка 🔴→варево → подпитка, 📖 мигает → тап → книга
9. Свободная игра далее — без подсказок
10. Второй ран: обучение не повторяется
11. Кнопка «Пропустить» → обучение заканчивается сразу
```

## B5 — Juice (часть 1): анимации

**Цель**: все действия имеют приятную визуальную отдачу.
Merge, бросок, попадание, смерть — всё анимировано.

### Фичи

1. **Tween-система**
   - `TweenRunner`: MonoBehaviour, список активных tweens,
     обновление в Update
   - Типы: `TweenPosition`, `TweenScale`, `TweenAlpha`,
     `TweenFloat`, `TweenSequence`, `TweenParallel`
   - Ease-функции: Linear, EaseInOut, EaseOutElastic,
     EaseOutBounce, EaseOutBack
   - API: `Tween.Position(transform, target, duration, ease)`

2. **Анимации перегонки**
   - Merge: ингредиенты притягиваются (lerp 0.2s) →
     вспышка → shockwave (ring SDF) →
     варево bounce-in (EaseOutElastic 0.3s)
   - Подпитка: ингредиент летит к вареву (0.15s) →
     glow flash → жидкость поднимается (0.2s)
   - Сбор в инвентарь: варева летят к слотам
     (bezier curve, stagger 0.1s) → bounce при приземлении
   - Растворение оставшихся ингредиентов: fade out с частицами

3. **Анимации боя**
   - Бросок зелья: слот пульсирует → «снаряд» летит
     по параболе (0.25s) → взрыв / эффект
   - Попадание: враг мигает белым (0.05s) → shake (3px, 0.1s)
   - Число урона: появляется над врагом, bounce вверх (0.6s),
     fade out
   - Движение игрока: squash & stretch (сжатие в направлении
     движения)
   - Движение врага: аналогичный squash & stretch
   - Толчок: враг «отлетает» (быстрый lerp + bounce at wall)

### Приёмочный тест (Play Mode)

```
1. Merge: два ингредиента плавно слетаются, вспышка,
   кольцо-шоквейв, варево «пружинит» при появлении
2. Подпитка: ингредиент влетает в варево, жидкость
   плавно поднимается
3. Сбор: варева по очереди взлетают в слоты зелий
4. Бросок: «снаряд» летит дугой от персонажа к клетке
5. Попадание: белая вспышка на враге, число урона
   «подпрыгивает» над ним
6. Движение: персонаж слегка сплющивается в направлении хода
7. Толчок: враг быстро отлетает, при ударе о стену — squash
8. Все анимации плавные (60 FPS), нет frame drops
```

## B6 — Juice (часть 2): эффекты и пост-процесс

**Цель**: combo-эффекты, dissolve, slowmo, полный пост-процессинг.

### Фичи

1. **Combo juice**
   - При combo: `ShaderGlobalController.SetSlowmo(0.05, 0.15s)` →
     мир замедляется, эффект усиленный
   - Текст «COMBO» (scale up + glow, золотой) → fade out
   - Расширенные VFX: ×2 количество частиц, ×1.5 размер
   - Screenshake усиленный (8px, 0.25s)
   - Fullscreen flash (белый, 0.15s)

2. **Dissolve при смерти**
   - `SH_Dissolve` (функция в EntityCommon.cginc):
     noise threshold → discard fragments + emission edge
   - TweenFloat: `_DissolveProgress` 0→1 за 0.4s
   - Частицы разлетаются от позиции врага (8–12 частиц, radial)
   - «Звёздочки опыта» летят к игроку (bezier curve)

3. **Полный пост-процессинг**
   - `SH_PP_Final`: виньетка (всегда) + хроматическая аберрация
     (при ударе, 0.1s) + damage vignette (красная, при уроне)
   - `SH_FX_DamageNumber`: 7-segment SDF, bounce ease,
     тёмный контур для читаемости
   - Hitstop: `Time.timeScale = 0.05` на 0.05s при каждом
     попадании зельем

### Приёмочный тест (Play Mode)

```
1. Combo: Молния в воду → мир замедляется на миг →
   усиленная молния → «COMBO!» золотым → screenshake
2. Убить врага → dissolve (фрагменты исчезают с яркой каймой) →
   частицы разлетаются → звёздочки летят к игроку
3. Получить урон → красная вспышка по краям экрана,
   хроматическая аберрация на 0.1s
4. Попадание зельем → мир на миг застывает (hitstop),
   затем числа урона с bounce
5. Виньетка видна всегда (края темнее центра)
6. Все эффекты не роняют FPS ниже 55
```

## B7 — Баланс, качество, книга

**Цель**: игра сбалансирована, есть fallback для слабых устройств,
книга алхимика полная.

### Фичи

1. **Баланс**
   - Проверить таблицу урона (GDD секция 18):
     все значения в DamageService соответствуют таблице
   - Прогон BalanceSimulator: 1000 ранов, целевой win rate 35–45%
   - Если win rate вне диапазона: скорректировать HP врагов,
     урон зелий, количество врагов
   - Ручные плейтесты: 5+ человек, 3 сессии каждый
   - Фиксация на основе обратной связи

2. **Система качества и fallback**
   - `QualityController`: автоопределение
     (GPU level, device type, runtime FPS)
   - `QUALITY_HIGH` / `QUALITY_LOW` keywords
   - LOW: noise из LUT, fbm 2 октавы, без каустики,
     без хром. аберрации, bloom в 2 pass
   - Настройка screenshake в Settings (слайдер 0–100%)

3. **Книга алхимика и настройки**
   - `BookScreen`: полная — все рецепты, карточки с описанием,
     AoE-визуализация, урон по уровням
   - Доступ: из меню + из боя (кнопка 📖)
   - Неразблокированные: «?» с подсказкой условия
   - Экран настроек: SFX, музыка, вибрация, screenshake,
     подтверждение хода, сброс прогресса
   - Сохранение настроек в PlayerPrefs

### Приёмочный тест (Play Mode)

```
1. Симулятор: 1000 ранов, win rate в логе.
   Если 35–45% — ОК
2. На слабом устройстве (или эмуляция LOW): всё читаемо,
   ≥50 FPS, bloom есть но попроще
3. Книга: все 15 зелий, тап → карточка с описанием,
   AoE-паттерн, урон по уровням
4. Неразблокированные рецепты показывают «?»
   и условие разблокировки
5. Настройки: screenshake = 0% → нет тряски,
   вибрация выкл → нет вибрации
6. Сброс прогресса → подтверждение → всё обнулено
7. 5 плейтестеров: средний ран 8–12 минут,
   никто не stuck, fun factor ≥ 7/10
```
