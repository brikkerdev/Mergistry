# Alpha — Итерации

> **Цель фазы**: полный ран от начала до финального босса.
> Все типы контента, карта с развилками, события, реликвии, 3 этажа.
>
> **Предусловие**: MVP завершён и стабилен.
>
> **Длительность**: 5–6 недель (8 итераций × 3–5 дней)

## A1 — Все элементы и рецепты

**Цель**: на доске 5 элементов, все 15 зелий создаются корректно.

### Фичи

1. **Элементы Lux и Umbra**
   - Добавить в `ElementType`, цвета, placeholder-шейдеры
     (жёлтый круг, фиолетовый круг)
   - `DistillationService.GenerateBoard`: пул элементов зависит
     от этажа (3 / 4 / 5)
   - Базовые зелья: Сияние (🟡+🟡), Мрак (🟣+🟣) — AoE,
     урон, эффекты

2. **Все рецептурные зелья**
   - Все 10 рецептов в `RecipeDB` (ScriptableObject)
   - `PotionDefinition` SO для каждого: базовый урон, AoE-паттерн,
     эффект, описание
   - `DamageService.GetAffectedCells`: все AoE-паттерны
     (крест, линия, облако, область, одиночная)
   - Разные эффекты зелий: урон, DoT, оглушение, ослепление,
     замедление, телепорт, проклятие, хаос

3. **Книга рецептов (базовая)**
   - `BookScreen`: список всех рецептов, разделённый на базовые
     и рецептурные
   - Карточка зелья: название, рецепт, AoE, урон по уровням,
     описание эффекта
   - Кнопка 📖 в HUD → открытие книги
   - Неразблокированные рецепты: «?» (подготовка к мета-прогрессии,
     но пока все открыты)

### Приёмочный тест (Play Mode)

```
1. Начать ран → доска с 3 элементами (этаж 1)
2. Создать Пламя, Поток, Отраву — базовые работают
3. Создать Пар (🔴+🔵), Напалм (🔴+🟢), Кислоту (🔵+🟢) —
   рецептурные работают
4. Открыть книгу (📖) → все 15 зелий видны с описаниями
5. Принудительно перейти на этаж 2 (debug) →
   на доске 4 элемента (+🟡)
6. Создать Молнию (🔵+🟡) → линейный AoE работает
7. На этаже 3 → 5 элементов, создать Хаос (🟡+🟣) →
   случайный эффект
8. Каждое зелье наносит правильный урон (проверить
   по таблице GDD)
```

## A2 — Враги (часть 1)

**Цель**: Гриб-бомба, Магнит-голем и Бронированный жук
работают по правилам.

### Фичи

1. **Гриб-бомба**
   - `MushroomBombBehavior`: таймер 3→2→1→💥, взрыв 3×3, урон 5
   - `EnemyView`: placeholder SDF (полукруг + прямоугольник),
     цвет меняется (жёлтый → красный по таймеру)
   - Число-таймер над головой
   - Взрыв наносит урон и врагам (можно использовать тактически)

2. **Магнит-голем и Бронированный жук**
   - `MagnetGolemBehavior`: притяжение игрока на 1 клетку,
     контактный урон 2, HP 6
   - `ArmoredBeetleBehavior`: как скелет + система брони
   - `DamageService.ApplyDamage`: обработка брони
     (armorPoints поглощают урон первыми)
   - Визуал брони: индикатор (ромбики над врагом),
     гаснут при снятии
   - Эффект Кислоты: снятие брони (armorPoints = 0)

3. **Толчок врага**
   - `CombatService.PushEnemy`: свайп от врага на соседней клетке
   - Враг сдвигается на 1 клетку в направлении толчка
   - Если за врагом стена → +1 урон
   - Анимация толчка: tween позиции + squash
   - InputService: распознавание свайпа от врага (vs свайп от игрока)

### Приёмочный тест (Play Mode)

```
1. Бой с 1 грибом-бомбой + 2 скелетами:
    - Гриб показывает «3» → «2» → «1» → взрыв 3×3
    - Скелеты в зоне взрыва получают 5 урона и умирают
    - Игрок в зоне взрыва получает 5 урона
2. Бой с магнит-големом:
    - Каждый ход игрока притягивает на 1 клетку к голему
    - При контакте — 2 урона
    - Убить зельем lv2+ (HP = 6)
3. Бой с бронированным жуком:
    - Бросить Пламя lv1 (3 урона) → жук потерял 2 брони + 1 HP
    - Бросить Кислоту → броня снята, теперь полный урон
4. Толкнуть скелета → сдвинулся на 1 клетку
5. Толкнуть скелета в стену → сдвинулся + 1 доп. урон
```

## A3 — Враги (часть 2) и зоны

**Цель**: Зеркальный слайм, Фантом, Некромант работают.
Зоны на поле (огонь, вода, яд) функционируют.

### Фичи

1. **Зеркальный слайм, Фантом, Некромант**
   - `MirrorSlimeBehavior`: копирует последнее зелье игрока
     (из CombatModel.combatLog), бросает обратно
   - `PhantomBehavior`: телепортация на случайную клетку
     в радиусе 2 от игрока, атака
   - `NecromancerBehavior`: воскрешение убитого врага (1 HP),
     движение от игрока
   - Placeholder SDF для каждого

2. **Зоны на поле**
   - `ZoneInstance`: тип, позиция, turnsRemaining
   - `GridModel.cells[].zones`: список зон на клетке
   - Зелья создают зоны: Пламя → огонь (2 хода),
     Поток → вода (3 хода), Отрава → яд (3 хода)
   - `CombatService.EndTurn`: тик зон (decrement duration),
     применение zone damage (1 урон огонь/яд за ход)
   - `ZoneOverlayView`: полупрозрачный цветной квадрат
     на тайле (placeholder)
   - Зоны влияют на **всех** (игрока тоже)

3. **Статус-эффекты**
   - `StatusEffect`: тип + duration
   - Замедление (движение −1): от паутины, воды
   - Оглушение (пропуск хода): от Вспышки
   - Отравление (DoT): от Отравы, Напалма
   - `CombatService.EndTurn`: тик эффектов,
     применение DoT к отравленным

### Приёмочный тест (Play Mode)

```
1. Бой с зеркальным слаймом:
    - Бросить Пламя → слайм показывает «скопирую Пламя»
    - Ход врагов → слайм бросает Пламя по игроку
    - Бросить Пар (0 урона) → слайм копирует Пар (безобидно)
2. Бой с фантомом:
    - Фантом показывает зону возможных позиций (кольцо вокруг игрока)
    - Ход врагов → телепортация + атака
3. Бой с некромантом + 2 скелетами:
    - Убить скелета → некромант воскрешает его с 1 HP
    - Убить некроманта → воскрешение прекращается
4. Бросить Пламя → зона огня (оранжевый оверлей) на 5 клетках
5. Враг входит в зону огня → получает 1 урон
6. Игрок входит в зону огня → тоже получает 1 урон
7. Через 2 хода зона огня исчезает
8. Бросить Отраву → зона яда + враг отравлен (DoT 1/ход)
```

## A4 — Карта и многоэтажность

**Цель**: между боями — карта этажа с развилками.
Три этажа, переход между ними.

### Фичи

1. **Генерация карты**
   - `MapGeneratorService.GenerateFloor`: алгоритм из архитектурного
     документа (3–4 ряда, развилки, 1 элита, 1–2 события, 1 босс)
   - `FloorMapModel`: узлы, связи, типы
   - `MapNodeView`: круги с иконками (⚔️ ⭐ ❓ 👑)
   - `MapPathView`: линии между узлами

2. **Навигация по карте**
   - `MapState`: отображение карты, обработка тапа на узел
   - Доступные узлы подсвечены, недоступные — тусклые
   - Тап на доступный узел → переход:
     ⚔️ → DistillationState → CombatState,
     ❓ → EventState (заглушка),
     ⭐ → DistillationState → CombatState (усиленные враги),
     👑 → DistillationState → BossState (заглушка)
   - После победы → возврат на карту, узел помечен как пройденный

3. **Три этажа**
   - `RunModel.currentFloor`: 0, 1, 2
   - Победа над боссом → следующий этаж (новая карта)
   - Этаж 3, босс побеждён → Victory (ResultScreen)
   - Составы врагов зависят от этажа
     (`MapGeneratorService.GenerateCombatSetup`)
   - Пул элементов на доске зависит от этажа (3 / 4 / 5)

### Приёмочный тест (Play Mode)

```
1. Меню → «Старт» → карта этажа 1:
   узлы с иконками, линии-пути, «ТЫ ЗДЕСЬ» внизу
2. Тап на ⚔️ узел → перегонка → бой → победа → карта
3. Пройденный узел тусклый с галочкой
4. Развилка: два узла доступны, выбрать ⭐ (элиту) →
   усиленный бой (3 врага)
5. Дойти до 👑 (босс) → бой (заглушка — обычные враги,
   сетка 5×5)
6. Победа → «Этаж 2» → новая карта, 4 элемента на доске
7. Пройти этаж 2 → этаж 3 → 5 элементов
8. Победить «босса» этажа 3 → Victory screen
```

## A5 — События и реликвии

**Цель**: события (фонтан, торговец, алтарь, сундук) работают.
После элиты — выбор реликвии.

### Фичи

1. **Четыре типа событий**
   - `EventState`: показ экрана события, обработка выбора
   - `EventScreen`: фон + описание + кнопки выбора
   - ⛲ Фонтан: [Выпить] 70% полное лечение / 30% −1 maxHP,
     [Наполнить] случайное зелье lv2
   - 🛒 Торговец: 3 зелья, цена 1–2 HP, купить одно
   - ⚗️ Алтарь: удалить зелье → случайная реликвия
   - 🎁 Сундук: выбор 1 из 3 (зелье / +3 HP / +1 действие)
   - `LootService.GenerateEventPotion`,
     `LootService.GenerateMerchantStock`

2. **Реликвии (5 штук)**
   - `RelicDefinition` SO: id, название, описание, эффект
   - `RelicManager`: список активных реликвий, проверка наличия
   - Реализовать 5 из 10: Термос, Линза, Фляга, Кубик, Призма
   - `RelicChoicePopup`: показ 3 реликвий после элиты,
     тап = выбор
   - Иконки реликвий в HUD (маленькие, рядом с HP)
   - Эффекты: Термос (25% не тратить кулдаун),
     Линза (AoE +1), Фляга (+1 HP после боя),
     Кубик (+1 действие перегонки), Призма (combo ×1.5)

3. **Интеграция реликвий в сервисы**
   - `DamageService`: проверка Линзы (расширить AoE),
     Призмы (×1.5 combo)
   - `CombatService.ThrowPotion`: проверка Термоса
     (шанс не тратить кулдаун)
   - `CombatService.EndCombat`: проверка Фляги (+1 HP)
   - `DistillationService`: проверка Кубика (+1 действие)

### Приёмочный тест (Play Mode)

```
1. На карте узел ❓ → тап → EventScreen «Фонтан»
2. [Выпить] → лечение (или потеря maxHP, 30%)
3. Торговец: 3 зелья с ценой, купить одно → HP уменьшается,
   зелье в инвентаре
4. Алтарь: удалить зелье → реликвия появляется в HUD
5. Сундук: выбрать +3 HP → HP увеличивается
6. Победить элиту → RelicChoicePopup:
   3 реликвии, тап → реликвия в HUD
7. С реликвией Фляга: после боя +1 HP
8. С реликвией Кубик: на перегонке 4 действия вместо 3
9. С реликвией Линза: AoE Пламени — 9 клеток вместо 5
```

## A6 — Боссы

**Цель**: три уникальных босса с двумя фазами каждый.

### Фичи

1. **Босс-фреймворк**
   - `BossState`: расширение CombatState
   - Сетка 6×6 для босс-боёв
   - `IBossBehavior`: интерфейс с поддержкой фаз
     (`DetermineIntent` получает текущую фазу)
   - Переключение фазы при HP ≤ 50%: визуальная отбивка
     (экран мигает, текст «Фаза 2»)
   - Босс занимает 2×2 клетки (4 клетки, все считаются occupant-ом)
   - Boss HP bar вверху экрана (горизонтальная полоса)

2. **Три босса**
   - Королева пауков (этаж 1): паутина + линейные атаки,
     фаза 2: призыв пауков + отступление
   - Железный голем (этаж 2): удар 2×2 + притяжение + броня 6,
     фаза 2: ломает столбы + ускорение
   - Алхимик-Отступник (этаж 3): копирование зелий + зоны,
     фаза 2: котлы генерируют зоны + телепорт
   - Placeholder SDF для каждого (увеличенные версии обычных врагов)
   - Интерактивные объекты: столбы (непроходимые),
     котлы (HP 3, уничтожаемые)

3. **Столбы и котлы**
   - `InteractableModel`: позиция, HP (для котлов),
     isPassable (false для столбов)
   - Зелья могут попадать в котлы → наносят урон
   - Уничтожение столба (босс голем, фаза 2): AoE осколков 2×2
   - Уничтожение котла (Алхимик): лишает телепортации

### Приёмочный тест (Play Mode)

```
1. Дойти до босса этажа 1 → сетка 6×6,
   Королева пауков (2×2) + 3 клетки с паутиной
2. Фаза 1: каждый ход 2 линии паутины,
   движение к игроку
3. Снизить HP до 50% → «Фаза 2»,
   призываются 2 паука, королева отступает
4. Убить → переход на этаж 2
5. Босс этажа 2: Железный голем, 2 столба на поле
6. Фаза 1: удар 2×2, притяжение, столбы блокируют
7. Фаза 2: столбы ломаются (AoE), голем ускоряется
8. Босс этажа 3: Алхимик, 3 котла
9. Уничтожить котлы → телепортация прекращается
10. Убить финального босса → Victory screen
```

## A7 — Combo и модификаторы комнат

**Цель**: зелья взаимодействуют с зонами (combo).
Комнаты имеют модификаторы.

### Фичи

1. **Combo-система (3 из 7)**
   - `DamageService.CheckCombo`: проверка зелья + зоны
     на целевой клетке
   - Молния + Вода: ×2 урон всем на воде
   - Огонь + Яд: облако яда расширяется
   - Поток + Лёд: заморозка врагов на воде
   - Лёд создаётся при Поток + Лёд combo (пока единственный
     источник льда)
   - При combo: screenshake (5px, 0.2s) + текст «COMBO»
     (scale up → fade)

2. **Модификаторы комнат (3 из 6)**
   - `RoomModifier` в `CombatSetup`
   - 🌊 Затопленная: вся сетка = вода (молния ×2, огонь ×0.5)
   - 🔥 Горящая: 1–2 случайные клетки загораются каждый ход
   - 🕳 С ямами: 2–3 непроходимые клетки,
     толчок врага в яму = мгновенная смерть
   - Иконка модификатора на узле карты
   - `CombatService.EndTurn`: логика горящей комнаты
   - `CombatService.PushEnemy`: проверка ямы

3. **Зона льда**
   - `ZoneType.Ice`: скольжение (существо проезжает +1 клетку)
   - Применяется при движении через зону льда
   - Placeholder визуал (голубой оверлей)

### Приёмочный тест (Play Mode)

```
1. Бросить Поток → зона воды на 4 клетках
2. Бросить Молнию в клетку с водой → «COMBO!»,
   все враги на воде получают ×2 урон
3. Бросить Пламя в клетку с ядом → облако яда расширяется
4. Комната 🌊 (затопленная): вся сетка — вода,
   Молния ×2 ко всем
5. Комната 🔥 (горящая): в конце хода 1–2 клетки загораются
6. Комната 🕳 (с ямами): 2 чёрные клетки,
   толкнуть врага в яму → мгновенная смерть
7. Зона льда: игрок входит → проезжает на 1 клетку дальше
```

## A8 — Полные шейдеры и сохранение

**Цель**: все сущности имеют полные SDF-шейдеры с наполнением.
Bloom работает. Прогресс сохраняется.

### Фичи

1. **SDF-шейдеры сущностей (полные)**
   - `SH_Player`: flow noise внутри плаща, цвет последнего зелья
   - `SH_Skeleton`: мерцающие глаза (sin pulse), рёбра
   - `SH_Spider`: анимированные ноги (sin per leg)
   - `SH_MushroomBomb`: lerp цвета по таймеру, ускоряющаяся пульсация
   - `SH_MagnetGolem`: voronoi текстура, волны притяжения (ring SDF)
   - `SH_MirrorSlime`: metaballs + HSV hue shift
   - `SH_ArmoredBeetle`: шестиугольник + specular
   - `SH_Phantom`: мерцающая alpha + trail
   - `SH_Necromancer`: спиральный вихрь + руны
   - `SH_Ingredient`: заполнение по типу элемента
     (fBm для огня, волны для воды и т.д.)
   - `SH_Brew`: колба с жидкостью, пузырьки, уровень = высота
     заполнения
   - Зонные шейдеры: `SH_Zone_Fire`, `SH_Zone_Water`,
     `SH_Zone_Poison`, `SH_Zone_Ice`

2. **Bloom пост-процессинг**
   - `SH_PP_BloomExtract`, `SH_PP_BloomBlur`, `SH_PP_BloomCombine`
   - `PostProcessController` на камере: OnRenderImage pipeline
   - Bloom threshold = 0.8, intensity = 1.5
   - Все glow-элементы (зелья, варева, эффекты) ярче порога → bloom

3. **Система сохранения**
   - `SaveService`: SaveMeta, LoadMeta, SaveRun, LoadRun
   - `meta.json`: пока минимальный (stats, settings)
   - `run.json`: mid-run save (при OnApplicationPause,
     после каждого боя)
   - Загрузка при старте: если есть run.json → popup
     «Продолжить прерванный ран?»
   - Удаление run.json при завершении рана

### Приёмочный тест (Play Mode)

```
1. Все сущности визуально отличимы и анимированы:
    - Огненные ингредиенты мерцают
    - Водные ингредиенты волнятся
    - Варева — колбы с жидкостью, уровень виден
    - Глаза скелета мерцают
    - Ноги паука шевелятся
    - Слайм переливается
    - Фантом мерцает
    - Голем имеет волны притяжения
2. Bloom: все светящиеся объекты имеют ореол свечения
3. Зоны на поле: огонь = анимированное пламя (noise),
   вода = волны, яд = пузырьки
4. Закрыть приложение mid-run → открыть → popup →
   «Продолжить» → ран продолжается с того же места
5. Завершить ран → run.json удалён
```
