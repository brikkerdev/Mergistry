# MVP — Итерации

> **Цель фазы**: играбельный прототип — один этаж, 3 боя линейно,
> merge работает, зелья наносят урон, можно выиграть или умереть.
>
> **Длительность**: 4–5 недель (6 итераций × 3–5 дней)

---

## M1 — Фундамент и доска

**Цель**: на экране доска 4×4, ингредиенты можно перетаскивать.

### Фичи

1. **Инфраструктура загрузки**
   - Boot-сцена, Game-сцена (аддитивная загрузка)
   - ServiceLocator с регистрацией пустых заглушек сервисов
   - EventBus (publish / subscribe / unsubscribe)
   - GameStateMachine с двумя состояниями: `MenuState` (кнопка «Старт»)
     и `DistillationState`

2. **Доска перегонки — модель и вид**
   - `BoardModel`: сетка 4×4, `CellContent` (type, elementType)
   - `DistillationService.GenerateBoard`: заполнение 3 элементами
     (Ignis, Aqua, Toxin), равномерное распределение, seed
   - `BoardView`: 16 quad-ов, цветные круги (placeholder — unlit
     solid color material)
   - `IngredientView`: позиция привязана к клетке, idle-покачивание
     (sin по Y)

3. **Drag & Drop ингредиентов**
   - `InputService`: распознавание DragStart / DragUpdate / DragEnd
   - `BoardView`: определение ингредиента под пальцем
     (worldPos → gridPos), перемещение визуала за пальцем
   - Подсветка валидных соседних клеток (зелёная рамка)
   - Bounce-back при отпускании вне валидной цели
   - Счётчик действий (3 точки, не уменьшается — merge ещё нет)

### Приёмочный тест (Play Mode)

```
1. Запустить → экран меню с кнопкой «Старт»
2. Нажать «Старт» → доска 4×4 с цветными кругами (3 цвета)
3. Зажать палец на ингредиенте → увеличивается, соседние
   клетки подсвечиваются
4. Перетащить на соседнюю клетку → bounce-back
   (merge ещё не работает)
5. Отпустить вне цели → вернулся на место
6. Три точки действий видны, не уменьшаются
```

## M2 — Merge, подпитка, инвентарь

**Цель**: можно мержить ингредиенты, получать варева,
подпитывать их, собирать в инвентарь.

### Фичи

1. **Merge-логика**
   - `DistillationService.CanMerge` и `PerformMerge`
   - Рецепты: 3 базовых (🔴+🔴, 🔵+🔵, 🟢+🟢)
     и 3 рецептурных (Пар, Напалм, Кислота)
   - При merge: оба ингредиента потребляются, варево lv1 появляется
     в клетке назначения, пустая клетка в источнике
   - `BrewView`: визуально отличается от ингредиента
     (прямоугольник вместо круга, другой цвет бордюра)
   - Подпитка: `CanInfuse` и `PerformInfuse` — ингредиент → соседнее
     варево, +1 уровень, валидация элемента
   - Счётчик действий уменьшается при merge и подпитке

2. **Инвентарь зелий**
   - `InventoryModel`: 4 слота, PotionSlot (type, level, cooldown)
   - `PotionSlotView`: 4 прямоугольника внизу экрана, цвет зелья,
     точки уровня
   - `DistillationService.CollectBrews`: при завершении фазы —
     все варева → инвентарь
   - Кнопка «В бой →» (завершает фазу досрочно)

3. **Обработка конфликтов инвентаря**
   - Если зелье того же типа уже есть — обновить уровень (если выше)
   - Если нет свободного слота — popup выбора замены
     (SlotReplacePopup: показать все слоты + новое зелье,
     тап = заменить)

### Приёмочный тест (Play Mode)

```
1. Доска 4×4, три элемента
2. Перетащить 🔴 на соседний 🔴 →
    - Оба исчезают
    - В клетке назначения — варево «Пламя lv1» (другой вид)
    - Клетка источника пустая
    - Точка действия погасла (осталось 2)
3. Перетащить 🔴 на варево «Пламя» →
    - Ингредиент исчезает
    - Варево стало «Пламя lv2» (2 точки)
    - Осталось 1 действие
4. Выполнить ещё один merge (🔵+🟢 = Кислота lv1)
    - Осталось 0 действий
5. Кнопка «В бой →» нажать →
   варева летят в слоты инвентаря →
   слоты показывают: Пламя lv2, Кислота lv1, пусто, пусто
6. Повторить с полным инвентарём →
   popup выбора замены появляется
```

## M3 — Боевая сетка и движение

**Цель**: после перегонки появляется сетка 5×5 с персонажем,
который двигается свайпами.

### Фичи

1. **Боевая сетка**
   - `CombatModel`: `GridModel` 5×5, `PlayerCombatModel`
   - `CombatService.InitCombat`: создание модели, расстановка игрока
     в (1,1)
   - `GridView`: 25 квадратов (placeholder — тёмно-серые quad-ы
     с тонкими линиями-разделителями)
   - `PlayerView`: яркий круг (placeholder) в клетке

2. **Движение игрока**
   - `InputService`: распознавание Swipe (от персонажа)
   - `CombatService.GetValidMoves`: все клетки в радиусе 2
     (горизонталь + вертикаль + L-форма), не через стены
   - Призрак (полупрозрачный дубликат) показывает конечную позицию
     до отпускания
   - `PlayerView`: tween-перемещение к целевой клетке (0.2s, lerp)
   - `CombatModel.player.hasMoved = true` после движения

3. **Переход Distillation → Combat**
   - `GameStateMachine`: переход `DistillationState → CombatState`
   - Простой fade (0.2s чёрный экран)
   - Доска скрывается, сетка появляется
   - HUD: слоты зелий остаются видны

### Приёмочный тест (Play Mode)

```
1. Меню → перегонка → сделать merge → «В бой →»
2. Экран темнеет → появляется сетка 5×5 с персонажем в (1,1)
3. Свайп вправо от персонажа → призрак в (2,1) или (3,1)
4. Отпустить → персонаж плавно перемещается
5. Свайп вверх → перемещение вертикально
6. Слоты зелий видны внизу, с зельями из перегонки
7. Повторный свайп → персонаж не двигается
   (уже ходил в этом ходу) [пока нет врагов — ход не кончается,
   это ОК, будет доработано в M5]
```

## M4 — Бросок зелий и кулдауны

**Цель**: можно выбрать зелье, бросить в клетку,
увидеть эффект, зелье уходит в кулдаун.

### Фичи

1. **Выбор и бросок зелья**
   - Тап на `PotionSlotView` → слот выделяется (яркий контур),
     на сетке подсвечиваются допустимые клетки (синий оверлей)
   - `DamageService.GetAffectedCells`: вычисление AoE
     (крест для Пламени, линия для Потока, 2×2 для Отравы)
   - Тап на допустимую клетку → `CombatService.ThrowPotion`
   - Отображение AoE-зоны (красный оверлей на affected cells)
     при наведении перед подтверждением
   - `EventBus.Publish(PotionThrown)` → `EffectView` (цветная
     вспышка в клетке, placeholder — расширяющийся круг
     цвета зелья, fade out за 0.3s)

2. **Кулдаун**
   - После броска: `slot.cooldownRemaining = 2` (или 1 для lv3)
   - `PotionSlotView`: затемнение + число ходов
   - Тап на зелье в кулдауне — ничего не происходит
     (слот не подсвечивается)
   - Тик кулдауна: пока вручную (кнопка «следующий ход»
     уменьшает кулдауны)

3. **Кнопка пропуска хода**
   - Кнопка ⏭️ под сеткой
   - Нажатие: `CombatService.SkipTurn` → +1 HP (если < max),
     все кулдауны −1
   - Сброс `hasMoved` и `hasActed` (новый ход)

### Приёмочный тест (Play Mode)

```
1. Перегонка: создать Пламя lv1 и Поток lv1
2. В бою: тап на слот Пламя → слот выделяется,
   клетки в радиусе 3 подсвечиваются синим
3. Тап на клетку → вспышка в крестообразной зоне (5 клеток),
   Пламя тускнеет, число «2» на слоте
4. Тап на Пламя → ничего (кулдаун)
5. Тап на Поток → подсвечивается, тап на клетку → линейный эффект
6. Нажать ⏭️ → кулдауны: Пламя = 1, Поток = 1
7. Нажать ⏭️ → кулдауны: Пламя = 0 (готово), Поток = 0 (готово)
8. Пламя снова доступно для броска
```

## M5 — Враги и пошаговый бой

**Цель**: на сетке есть скелет и паук, они ходят по правилам,
можно убить зельем, можно получить урон.

### Фичи

1. **Скелет**
   - `EnemyCombatModel`: entityId, position, HP=3, definition
   - `SkeletonBehavior`: определение intent-а
     (если рядом → ⚔️ atk 2, иначе → 🚶 move к игроку)
   - `EnemyView`: placeholder (белый квадрат с «глазами» —
     две маленькие точки)
   - `IntentionIconView`: иконка над врагом (⚔️ или →) +
     красная подсветка целевых клеток

2. **Паук**
   - `SpiderBehavior`: определение линии к игроку,
     intent = ⚔️ по линии (дальность 3), урон 1
   - `EnemyView`: placeholder (серый овал)
   - Красная подсветка линии атаки

3. **Пошаговый цикл и урон**
   - `CombatState` подсостояния:
     `WaitMove → WaitAction → AnimatePlayer → ResolveEnemy → Check`
   - `DamageService.ApplyDamage`: урон по врагу (HP -= damage,
     с учётом уровня зелья), смерть при HP ≤ 0
   - `DamageService.ApplyDamageToPlayer`: урон игроку
   - При попадании зельем: враг мигает белым (0.1s)
   - При смерти врага: враг исчезает (fade out 0.3s)
   - `AIService.DetermineIntents`: вызывается в конце каждого хода,
     обновляет intent-ы
   - `AIService.ExecuteIntents`: выполняет intent-ы всех врагов
   - Порядок хода: движение игрока → действие игрока →
     ходы всех врагов → тик кулдаунов → новый ход

### Приёмочный тест (Play Mode)

```
1. Перегонка: создать Пламя lv1
2. Бой: на сетке 1 скелет (3,3) и 1 паук (4,1)
3. Скелет показывает → (идёт к игроку),
   паук показывает ⚔️ + красная горизонтальная линия
4. Сдвинуться с линии паука (свайп вверх)
5. Бросить Пламя в скелета → скелет мигает, HP -= 3, скелет умирает
6. Нажать ⏭️ → ход врагов: паук стреляет (промахивается,
   игрока нет на линии)
7. Дождаться кулдауна Пламени → бросить в паука → паук умирает
8. Если встать на линию паука и пропустить ход →
   паук стреляет, игрок получает 1 урон
```

## M6 — Игровой цикл и смерть

**Цель**: полный цикл: меню → перегонка → бой → перегонка → бой →
перегонка → бой → результат.

### Фичи

1. **HP-дисплей и смерть**
   - `HealthBarView`: 5 сердечек (placeholder — красные/серые круги)
   - Обновление при `PlayerDamaged`: сердечко гаснет, красная
     вспышка по краям экрана (0.2s)
   - HP ≤ 0 → `CombatEnded(Defeat)` → переход в `ResultState`
   - `ResultScreen`: «Вы пали на Этаже 1, бой 2», кнопки
     «Ещё раз» и «Меню»

2. **Линейная структура рана (3 боя)**
   - `RunModel`: currentFight (0, 1, 2), HP сохраняется между боями
   - Последовательность: Distill → Fight1 (1 скелет) →
     Distill → Fight2 (2 скелета) →
     Distill → Fight3 (1 скелет + 1 паук) → Victory
   - `CombatEnded(Victory)` → если currentFight < 2:
     следующая перегонка → следующий бой
   - Если currentFight == 2 → `ResultScreen` с победой
   - Между боями HP **не восстанавливается**

3. **Базовые SDF-шейдеры**
   - `SH_Background`: вертикальный gradient (тёмно-синий)
   - `SH_Floor`: solid color + тонкие линии сетки
   - `SH_Player`: SDF круг + треугольник (плащ), solid color
   - `SH_Skeleton`: SDF прямоугольник + круг, точки-глаза
   - `SH_Spider`: SDF овал + 8 линий-ног
   - `SH_Ingredient`: SDF круг, цвет по элементу
   - `SH_Brew`: SDF прямоугольник + круг (колба), цвет зелья
   - `AlchemistSDF.cginc`: базовые SDF-примитивы
     (sdCircle, sdBox, sdTriangle, sdSegment, opUnion)

### Приёмочный тест (Play Mode)

```
1. Меню → «Старт» → перегонка (доска 4×4)
2. Сделать merge → зелья в инвентаре → «В бой»
3. Бой 1: 1 скелет → убить → победа
4. Автоматически → перегонка → «В бой»
5. Бой 2: 2 скелета → убить обоих → победа
6. Перегонка → «В бой»
7. Бой 3: 1 скелет + 1 паук → убить → победа →
   «Вы победили!», кнопка «Ещё раз»
8. Нажать «Ещё раз» → новый ран с начала
9. Альтернативно: получить урон, умереть →
   «Вы пали», кнопки «Ещё раз» / «Меню»
10. Все сущности рисуются SDF-шейдерами
    (геометрические фигуры, не квадраты)
```

## Чеклист завершения MVP

- [ ] Полный цикл: меню → перегонка → бой ×3 → результат
- [ ] Merge: одинаковые → базовое зелье, разные → рецептурное
- [ ] Подпитка: ингредиент → варево, +1 уровень, валидация элемента
- [ ] 3 действия за фазу перегонки, счётчик работает
- [ ] 3 базовых + 3 рецептурных зелья (Пламя, Поток, Отрава,
      Пар, Напалм, Кислота)
- [ ] Бросок зелья с AoE (крест, линия, 2×2)
- [ ] Кулдаун 2 хода (1 для lv3)
- [ ] Движение 0–2 клетки
- [ ] Пропуск хода (+1 HP)
- [ ] Скелет: идёт к игроку, атакует вблизи
- [ ] Паук: стоит на месте, стреляет по линии
- [ ] Намерения врагов отображаются
- [ ] HP 5, смерть, экран результата
- [ ] HP не восстанавливается между боями
- [ ] SDF-шейдеры для всех сущностей (силуэты)
- [ ] 60 FPS на целевом устройстве
